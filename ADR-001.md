# ADR 001: Arquitetura de Blog Estático de Alta Performance

## Status

**Proposto** (Data: 06 de Abril de 2026)

## Contexto

O objetivo é construir um blog utilizando **Next.js** que seja extremamente rápido, seguro e otimizado para o Google (SEO). A aplicação deve ser totalmente estática, sem dependência de APIs externas ou bancos de dados em tempo de execução, garantindo o melhor ranqueamento possível através de Core Web Vitals sólidos.

## Decisões Técnicas

### 1. Framework e Renderização

- **Next.js 14+ (App Router):** Utilização de _Server Components_ para minimizar o bundle de JavaScript enviado ao cliente.
- **Static Site Generation (SSG):** Todo o conteúdo será pré-renderizado em tempo de build. Isso garante um Time to First Byte (TTFB) mínimo e segurança contra vulnerabilidades de backend.

### 2. Estilização e UI

- **Styled Components:** Escolhido pela flexibilidade e escopo de CSS.
- **CSS-in-JS com SSR Registry:** Implementação obrigatória do `StyledComponentsRegistry` para injetar estilos críticos no servidor, evitando o _Cumulative Layout Shift_ (CLS).

### 3. Gestão de Conteúdo (Data Source)

- **Contentlayer:** Atuará como o motor de processamento. Ele transformará arquivos Markdown (`.md`) locais em dados JSON tipados.
- **TypeScript:** Garantia de contrato de dados. Se um post faltar um campo obrigatório (como a meta-descrição), o build falhará propositalmente para proteger o SEO.

### 4. SEO e Performance

- **Metadata API:** Gerenciamento centralizado de títulos, descrições e Open Graph tags.
- **Next/Image:** Otimização automática de imagens para formatos modernos (WebP/AVIF).
- **Next/Font:** Carregamento de fontes sem causar saltos de layout.

## Consequências

### Positivas

- **Performance:** Pontuação próxima de 100 em todos os pilares do Lighthouse.
- **Custo Zero/Baixo:** Pode ser hospedado em qualquer CDN (Vercel, Netlify, Cloudflare Pages).
- **SEO Nativo:** Conteúdo 100% visível para crawlers sem execução de JS no cliente.

### Negativas

- **Ciclo de Deploy:** Qualquer alteração no texto ou novo post exige um novo deploy (trigger via Git).
- **Configuração Inicial:** Exige um setup mais rigoroso de SSR para CSS-in-JS no App Router.

## Plano de Implantação

### Fase 1: Setup Estrutural

1. Inicialização do projeto com Next.js e TypeScript.
2. Configuração do `lib/registry.tsx` para suporte ao Styled Components.
3. Definição do `contentlayer.config.ts` com o esquema de posts.

### Fase 2: Desenvolvimento de UI e SEO

1. Criação de componentes semânticos (`<header>`, `<article>`, `<footer>`).
2. Implementação da Metadata API no `layout.tsx` e `page.tsx`.
3. Geração automática de `sitemap.ts` e `robots.ts`.

### Fase 3: Pipeline de Conteúdo

1. Criação da pasta `/content` para armazenamento dos arquivos `.md`.
2. Implementação de `generateStaticParams` nas rotas dinâmicas de posts.
3. Adição de Schema Markup (JSON-LD) para reconhecimento do Google.

### Fase 4: Produção

1. Auditoria de acessibilidade e Core Web Vitals.
2. Configuração de deploy contínuo (Vercel/GitHub).
3. Indexação inicial via Google Search Console.
